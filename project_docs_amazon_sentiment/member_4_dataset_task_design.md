# 成员 4：数据集与任务说明负责人

## 1. 角色定位

成员 4 是非代码成员，负责数据集介绍、任务定义和标签映射说明。

成员 4 的重点是解释项目为什么选择这个数据集，以及数据集如何支持两个分类任务。

## 2. 负责范围

成员 4 主要负责：

- 数据集来源说明。
- 数据字段说明。
- 三分类任务定义。
- 五星预测任务定义。
- 标签映射说明。
- 数据集相关 Q&A。

## 3. 输入材料

- 项目总览文档。
- `summary.json`。
- 成员 2 提供的数据统计表和图。

## 4. 具体工作步骤

1. 整理数据集基本信息。
   - 数据集名称：Amazon Reviews Multi / en。
   - 数据语言：英文。
   - 数据内容：Amazon 商品评论文本和星级评分。
   - 数据用途：商品评论情感分析和评分预测。

2. 说明数据集适配性。
   - 数据集中有评论文本，可以作为模型输入。
   - 数据集中有星级评分，可以作为五星预测标签。
   - 星级评分可以映射为三分类情感标签。
   - 数据已划分为 train、validation、test，适合模型训练和评估。

3. 说明字段含义。
   - `id`：评论编号。
   - `text`：评论文本。
   - `stars`：1-5 星评分。
   - `label_3class`：三分类情感标签。
   - `raw_label_5way`：原始 5 类标签。

4. 说明两个任务。
   - 三分类情感分析用于判断整体情绪。
   - 五星评分预测用于判断更细粒度的评价强度。
   - 两个任务来自同一数据集，但难度不同。

5. 说明标签映射。
   - 1-2 stars 映射为 `negative`。
   - 3 stars 映射为 `neutral`。
   - 4-5 stars 映射为 `positive`。
   - 该映射避免额外人工标注，并且符合商品评分的常见理解。

6. 准备数据集部分文字。
   - 写一段 report 中可使用的数据集介绍。
   - 写一页 PPT 可使用的数据集说明。
   - 准备 3-5 个数据集相关问答。
